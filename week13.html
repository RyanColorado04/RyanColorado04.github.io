I also worked on learning about non-strictness. I worked on a few examples I will add to GitHub which is linked to in my GitHub pages.

Non-strictness is a property of a function. It means that it may choose not to evaluate one or more of its arguments. A strict function always evaluates its arguments, and they are the norm in most programming languages. Most languages only support functions that that expect their arguments fully evaluated.

Strict example in Scala:


def square(x: Double): Double = x * x

Short circuiting Boolean functions && and || are an example of a non-strict function. The function && takes two Boolean arguments, but only evaluates the second argument if it is true.

Example:


false && { println(“!!”); true // does not print anything
